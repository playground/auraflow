AuraFlow — Flash your sensor

Browser-based ESP32 flashing for the AuraFlow water-flow leak detection sensor. Uses WebSerial — no toolchain install needed.

WebSerial isn't available in this browser. Use a recent Chrome or Edge on a desktop OS. iOS and Firefox are not supported.

1Connect the ESP32

Plug the ESP32 into your computer with a data USB cable (not a charge-only cable — a common gotcha). Hold the BOOT button on the board if your variant requires it for flashing.

2Flash the firmware

Click below, select your serial port (look for USB Serial or cp210x), and confirm. The flash takes ~30 seconds.

Browser doesn't support WebSerial — use Chrome or Edge. This page must be served over HTTPS (or localhost) for WebSerial to work.

3Provision over serial

After flashing, the ESP32 reboots and prints auraflow: NVS not provisioned until you give it Wi-Fi credentials and a HomeHub URL. Provisioning is currently done via the Moddable serial REPL, not from this page yet — see bring-up.md § 3 for the snippet to paste.

In short: open a serial terminal at 460800 8N1, wait for the prompt, then paste: 

import Preference from 'preference';
const D = 'auraflow';
Preference.set(D, 'wifiSsid',       'YourSSID');
Preference.set(D, 'wifiPassword',   'YourPassword');
Preference.set(D, 'homehubUrl',     'http://192.168.1.10:3000');
Preference.set(D, 'internalApiKey', 'paste-INTERNAL_API_KEY-here');
Preference.set(D, 'sensorId',       'auraflow-mainline-01');
Preference.set(D, 'wordOrder',      'low-word-first');

Reset the ESP32 and watch for wifi: gotIP.

4Register the sensor in HomeHub

  1. Open your HomeHub dashboard (http://<host>:4201) and sign in
  2. Navigate to Sensors → + Add sensor
  3. Use the same sensorId you wrote to NVS in step 3
  4. Within ~30 s the sensor's last-seen timestamp will tick
